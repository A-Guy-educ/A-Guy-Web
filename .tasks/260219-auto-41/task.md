# Task

**Description**

In dark mode, the design system relies heavily on very similar blue tones.
This creates low visual distinction between components, states, and hierarchy levels.

### Problem

* Multiple UI elements use nearly identical blue shades.
* Insufficient contrast between primary, secondary, and interactive states.
* Limited visual hierarchy.
* UI feels flat and visually repetitive in dark mode.

### Expected Outcome

* Introduce a broader and well-structured dark mode palette.
* Add complementary hues (not only blue) while preserving brand consistency.
* Ensure proper contrast and accessibility (WCAG compliant).
* Define clear roles for each color (primary, secondary, accent, success, warning, destructive, muted, etc.).
* Maintain harmony between shades and ensure they work together consistently.

### Task

1. Audit the current dark mode color tokens.
2. Identify redundant or overly similar blue shades.
3. Propose a revised palette with:

   * Clear tonal steps
   * Complementary accent colors
   * Proper contrast ratios
4. Validate combinations across core UI components.

Goal: A balanced, expressive, and accessible dark theme with clear visual hierarchy — not a monochromatic blue interface.
