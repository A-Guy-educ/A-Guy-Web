# fix: Image Upload Error Handling & Quality Constraints

## What

When users upload low-resolution or highly compressed images in the chat interface, they receive a generic 'Upload failed' error instead of an actionable message. The error handling defaults to retry 3 times on status=0 before showing generic error. Need to add client-side image dimension validation with actionable error messages and improve error extraction from Vercel Blob responses.

## Scope

- `src/ui/web/chat/hooks/useDirectChatAssetUpload.ts`
- `src/server/chat-assets/constants.ts`
- `src/app/api/chat-assets/finalize/route.ts`

**Type:** bugfix | **Risk:** medium

## Changes

Added client-side image dimension validation (100x100px minimum) before upload with actionable error messages, and improved error extraction from Vercel Blob responses to surface `serverMessage` property when available.
